# 🎬 مولد فيديوهات Shorts بالذكاء الاصطناعي

موقع جاهز للنشر — الزائر بيكتب فكرة بس ويطلعله فيديو، من غير ما يحط أي مفتاح API.
المفاتيح دي بتاعتك انت (صاحب الموقع) ومخفية في إعدادات السيرفر.

---

## 1) التشغيل على جهازك أول مرة

### تسطيب بايثون والمكتبات
```
python -m venv venv
```
تفعيل البيئة:
- Windows: `venv\Scripts\activate`
- Mac/Linux: `source venv/bin/activate`

```
pip install -r requirements.txt
```

هذا هيسطبلك: `streamlit` (الواجهة)، `google-generativeai` (Gemini)، `gTTS` (الصوت العربي)،
`moviepy` (المونتاج)، `requests` (تحميل الفيديوهات/الصور)، `Pillow`، و`imageio-ffmpeg`
(بيوفر FFmpeg تلقائياً بدون تسطيب يدوي).

### 2) حط مفاتيحك الخاصة (خطوة مهمة جداً)

الموقع بيقرا المفاتيح من ملف اسمه `secrets.toml` جوه مجلد `.streamlit/` — الملف ده
**مالكه انت بس** ومحدش من الزوار بيشوفه أو يقدر يوصله.

1. هتلاقي ملف اسمه `.streamlit/secrets.toml.example` — اعمل نسخة منه وسمّيها `secrets.toml`
   (نفس المجلد `.streamlit/`).
2. افتحه وحط فيه مفاتيحك الحقيقية:
```toml
GEMINI_API_KEY = "المفتاح_بتاعك"
PEXELS_API_KEY = "المفتاح_بتاعك"
```
3. مفتاح Gemini موجود عندك بالفعل. مفتاح Pexels مجاني تماماً، تقدر تاخده من
   [pexels.com/api](https://www.pexels.com/api/).

⚠️ **مهم:** ملف `secrets.toml` (بدون .example) متعمله رفع على GitHub أبداً — فيه
مفاتيحك الحقيقية. الملف `.gitignore` المرفق بيمنع ده تلقائياً.

### 3) شغّل الموقع محلياً للتجربة
```
streamlit run app.py
```
هيفتح على `http://localhost:8501`

---

## 2) نشر الموقع على النت عشان الناس تستخدمه فعلاً

أسهل وأسرع طريقة مجانية هي **Streamlit Community Cloud**:

1. ارفع مجلد المشروع (بدون ملف `secrets.toml` الحقيقي) على GitHub.
2. ادخل على [share.streamlit.io](https://share.streamlit.io) وسجّل بحساب GitHub.
3. اختار الـ repo بتاعك واضغط Deploy.
4. من إعدادات التطبيق (⋮ → Settings → Secrets)، الصق نفس محتوى `secrets.toml`
   بتاعك هناك — دي الطريقة الآمنة لحفظ مفاتيحك على السيرفر البعيد.
5. الموقع هيبقى شغال على رابط عام، وأي حد يفتحه هيكتب فكرته ويطلعله فيديو
   من غير ما يحتاج أي مفتاح.

بدائل تانية لو عايز تحكم أكتر (سيرفر خاص بيك): **Render** أو **Railway** — نفس
الفكرة، بس المفاتيح بتتحط كـ Environment Variables في إعدادات المشروع هناك بدل
ملف secrets.toml.

---

## 3) حماية نفسك من الاستخدام المبالغ فيه (مهم قبل ما تنشره)

بما إن كل فيديو بيتعمل بيكلفك فلوس فعلية من رصيد Gemini وPexels، الكود فيه بالفعل:
- **حد أقصى 3 فيديوهات لكل زائر في الجلسة الواحدة** (تقدر تغيّر الرقم في متغير
  `MAX_VIDEOS_PER_SESSION` جوه `app.py`).

نصايح إضافية قبل ما تفتحه للجمهور:
- تابع استهلاكك على Google AI Studio وPexels بشكل دوري.
- فكر تحط حد أقصى شهري كمان.

---

## 4) لما تيجي تخلّي الموقع مدفوع

ده خطوة تانية بعد ما يبقى الموقع شغال ومستقر. أشهر طريقة بسيطة هي إضافة نظام
اشتراك بسيط باستخدام **Stripe** (بوابة دفع) عن طريق:
- صفحة دفع بسيطة (Stripe Checkout) قبل ما الزائر يقدر يضغط "أنشئ الفيديو".
- أو نظام "أكواد وصول" (access codes) تديها للمشتركين بعد ما يدفعوا يدوياً.

الخطوة دي محتاجة تصميم إضافي (تسجيل دخول، تتبع مين دفع ومين لأ)، وممكن نبنيها
سوا في مرحلة لاحقة لما الموقع يبقى شغال ومجرّب كويس.

---

## استخدام الموقع (من ناحية الزائر)
1. يكتب فكرة الفيديو.
2. يختار: فيديو بمشاهد متحركة ولا ألبوم صور متقلبة.
3. يضغط "أنشئ الفيديو الآن" ويستنى (من دقيقة لـ 3 دقايق حسب سرعة النت).
4. الفيديو يظهر مباشرة مع زرار تحميل.

## استكشاف الأخطاء
- رسالة "الموقع لسه مش مظبوط" → معناها ملف `secrets.toml` مش موجود أو فاضي.
- مشكلة متعلقة بـ ffmpeg → `pip install --upgrade imageio-ffmpeg`
